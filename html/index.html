Projet morpion multijoueur

Objectifs

  1. Développer un jeu morpion en multijoueur
  2. Utiliser une architecture client-serveur avec sockets Java
  3. Créer une interface graphique avec de l'IHM
  4. intégrer un docker pour héberger le serveur

Présentation du projet

Utilisation de Git

Lors de ce projet nous avons utiliser Git. Git est un système de gestion de versions décentraliser utilisé pour suivre les modification apportées a un projet au fil du temps. Il permet de :
sauvegarder l'historique complet du code source,
de travailler a plusieur sur le même projet,
de créer des branches indépendantes pour développer des fonctionnalité sans affecté le code présent sur la branche main et de fusionner des branches.
Grâce a ces nombreuses fonctionnalité Git nous a permis de collaborer sur ce projet de manière efficace.

Utilisation de Docker

Docker est outil de virtualisation qui permet de créer des conteneurs, c'est a dire des environneemnt isolés. Contrairement à une machine virtuelle, un conteneur est plus léger, rapide et facile à déployer.
Nous avons utilisé un docker dans notre projet pour héberger le serveur et uniformiser l'environnement d'exécuion.

Fonctionnement du jeu

  1. Le serveur attends la connection de deux joueurs
  2. Les clients jouent a tour de rôle
  3. le plateau est mis a jours après chaque tour
  4. les coups sont vérifier

Comment l'état du Morpion est-il communiqué a l'IHM?

Il sera nécessaire de communiquer l'entièreté du morpion.
Cette information sera communiquer ainsi :
"x" : croix
"o" : rond
"0" : case vide

Conclusion

Ce projet nous a permis de pratiquer la programmation réseau en java et de découvrir les dockers et git.
